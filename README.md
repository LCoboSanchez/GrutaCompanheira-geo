## How to reproduce

### 1. Get the data

Download the two data files from the [OSF project page](https://doi.org/10.17605/OSF.IO/2X3VB) and place them in the `data/` folder:

- `GDC_SPATIAL_FEB26_archmaterials.txt`
- `GDC_CLEAN_DATABASE_FEB_2026.xlsx`

### 2. Open the project in RStudio

Open `GDC_Spatial_Analysis.Rproj`. This sets all paths automatically ‚Äî no manual edits needed anywhere in the scripts.

### 3. Install required packages

**Option A ‚Äî exact versions used for the published analysis (recommended):**
```r
install.packages("renv")
renv::restore()
```

**Option B ‚Äî latest versions:**
```r
install.packages(c(
  "here",
  "spatstat", "spatstat.geom", "spatstat.explore", "spatstat.random",
  "sparr", "ggplot2", "patchwork", "dplyr", "tidyr",
  "flextable", "dixon", "readxl",
  "magick", "cowplot"   # needed for 03_composites.R only
))
```

### 4. Run the scripts in order

| Script | What it does | Run? |
|--------|-------------|------|
| `code/01_analysis.R` | Full spatial analysis ‚Äî all statistics, figures, and tables | **Required** |
| `code/02_replot.R` | Re-renders all figures with larger text for journal submission | Optional |
| `code/03_composites.R` | Assembles individual PDFs into multi-panel composite figures | Optional |

Run order: `01` ‚Üí `02` (optional) ‚Üí `03` (optional).

**Expected run time:** 10‚Äì20 minutes for `01_analysis.R`. Simulation-intensive steps use `nsim = 999` for Hopkins‚ÄìSkellam and scan tests, and `nsim = 39` for K-function envelopes (minimum achievable p = 0.001 and p = 0.025 respectively). All simulations use `set.seed(42)`.

> **Note on simulation reproducibility:** `set.seed(42)` is called before all K-function and NNequal envelope steps, so those envelope bands are exactly reproducible. The Hopkins‚ÄìSkellam and segregation tests use `spatstat`'s internal RNG; Monte Carlo p-values will be numerically similar but not bit-identical across platforms.

## Research questions

- **Q1** ‚Äî Spatial patterning of cementation types (TYPE_A / TYPE_B / TYPE_C / uncovered)
- **Q2** ‚Äî Spatial distribution of material types (BONE / LITHIC)
- **Q3** ‚Äî Association between cementation and material type (cross-tabulation)

All analyses use the `spatstat` ecosystem for R.

## Data description

### `GDC_SPATIAL_FEB26_archmaterials.txt`

Tab-separated text file. One row per recovered find. Columns:

| Column | Description |
|--------|-------------|
| `ID` | Find identifier |
| `x`, `y`, `z` | Absolute total-station coordinates (metres a.s.l.) |
| `level` | Stratigraphic level: `104_e`, `1`, `2`, `3`, `4` |
| `code` | Material type: `BONE` or `LITHIC` |
| `breccia_type` | Cementation type: `TYPE_A`, `TYPE_B`, `TYPE_C`, or blank/NA (= uncovered) |
| `date` | Recovery date |

Level 1 is excluded from analysis (late intrusive sediment). Level 4 = former field unit Level 1B; Level 3 = former field units Level 3 + Level 3/1B (merged by field team). Combined analysis uses levels `104_e`, `2`, `3`, and `4`.

### `GDC_CLEAN_DATABASE_FEB_2026.xlsx`

Full excavation database. Used here only to extract boundary reference coordinates (micromorphology block faces, cave topography points, profile-cleaning locations) that define the convex hull observation window (`win_xy`) for plan-view analyses.
